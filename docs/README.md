# OctoFit Tracker – Project Documentation

Welcome to the documentation for the **OctoFit Tracker** project, built with GitHub Copilot agent mode for Mergington High School. This README provides a high-level summary of the project management processes used by OctoAcme, and a directory of all documents in this folder.

---

## Project Management Process Summary

OctoAcme follows an iterative project management methodology inspired by Agile principles. The key phases are:

1. **Project Initiation** – Define the project goals, stakeholders, and success criteria. Identify the problem to be solved and secure initial resources and approvals.

2. **Planning** – Break the project into milestones and tasks, assign roles and responsibilities, establish timelines, and set up the development environment.

3. **Execution & Tracking** – Develop features iteratively, log activity and progress, conduct regular stand-ups, and track work against the plan.

4. **Risk Management & Communications** – Identify risks early, define mitigation strategies, and maintain clear communication channels with stakeholders throughout the project.

5. **Release & Deployment** – Prepare the application for release, complete testing, and deploy to the target environment (e.g., GitHub Codespaces or production).

6. **Retrospective & Continuous Improvement** – Review what went well and what could be improved after each iteration, and apply learnings to future cycles.

Roles, responsibilities, and stakeholder engagement are clearly defined at each phase to ensure accountability and transparency.

## About OctoFit Tracker

**OctoFit Tracker** is a fitness tracking web application developed for Mergington High School, built using:

- **Frontend**: React.js
- **Backend**: Python with Django REST Framework
- **Database**: MongoDB
- **Development Environment**: GitHub Codespaces with GitHub Copilot agent mode

Key features include user profiles, activity logging and tracking, team management, a competitive leaderboard, and personalized workout suggestions.
